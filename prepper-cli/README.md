# prepper-cli

`prepper-cli` is the standalone Python package that powers Prepper's OpenRouter chat and interview flows.

## What It Includes

```text
prepper-cli/
|- src/prepper_cli/main.py           # CLI entry point
|- src/prepper_cli/chat.py           # get_chat_reply()
|- src/prepper_cli/conversation.py   # in-memory conversation history
|- src/prepper_cli/interview.py      # interview flow + scoring
|- src/prepper_cli/benchmark.py      # simulated benchmark interviews
|- src/prepper_cli/system_prompts.py # prompt discovery/loading
`- src/prepper_cli/prompts/           # bundled prompts (.md)
```

## Setup

```bash
cd prepper-cli
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

Optional test deps:

```bash
pip install pytest
```

From the monorepo root, create root `.env` (or export env vars):

```bash
LLM_API_KEY=your_key_here
LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=openai/gpt-5.4
PREPPER_DEFAULT_SYSTEM_PROMPT=coding_focus
```

The older `OPENROUTER_API_KEY`, `OPENROUTER_BASE_URL`, and `OPENROUTER_MODEL` names are still supported as fallbacks.

For a local llama.cpp server, set:

```env
LLM_API_KEY=local-dummy
LLM_BASE_URL=http://127.0.0.1:8080/v1
LLM_MODEL=ministral
```

## CLI Usage

From the monorepo root, you can run the same CLI via `./prepper.sh --interactive` without manually activating `prepper-cli/.venv`.

### Interactive Mode (Default)

Running `prepper-cli` starts an interactive session and keeps conversation context in memory for that session.

```bash
prepper-cli
```

The monorepo wrapper uses `./prepper.sh --interactive` to route into prepper-cli, but the installed `prepper-cli` command starts interactive mode by default.

You can preselect an interview style:

```bash
prepper-cli --interview-style coding_focus
```

Or list available interview style IDs:

```bash
prepper-cli --list-interview-styles
```

Interview tuning flags:

```bash
prepper-cli --interview-style coding_focus --difficulty hard --question-limit 3 --pass-threshold 7.2
```

Model settings overrides:

```bash
prepper-cli --interview-style coding_focus --temperature 0.2 --top-p 0.9 --frequency-penalty 0.3 --presence-penalty -0.2 --max-tokens 500
```

Language + color:

```bash
prepper-cli --color --language de --interview-style behavioral_focus
prepper-cli --color --language fr --interview-style behavioral_focus
```

Exit with `exit` or `quit`.

## Benchmark Mode (`--benchmark`)

Benchmark mode simulates a full interview automatically.

- interviewer = selected `--interview-style`
- candidate = generated by a candidate simulation prompt
- final output includes candidate score and interviewer score

### Quick Examples

Default benchmark (strong candidate profile):

```bash
./prepper.sh --benchmark --interview-style behavioral_focus
./prepper.sh -b --interview-style behavioral_focus
```

Force weak candidate simulation:

```bash
./prepper.sh -b --interview-style behavioral_focus --weak-candidate
```

Short hard coding benchmark:

```bash
./prepper.sh -b --interview-style coding_focus --difficulty hard --question-limit 3
```

Override pass threshold:

```bash
./prepper.sh -b --interview-style coding_focus --pass-threshold 8.0
```

German and French benchmark:

```bash
./prepper.sh -b --interview-style behavioral_focus --language de
./prepper.sh -b --interview-style behavioral_focus --language fr
```

Use different models for runtime vs final benchmark scoring:

```bash
./prepper.sh -b --interview-style behavioral_focus --model openai/gpt-5.4 --benchmark-model openai/gpt-4.1
```

Print only the benchmark result as JSON, without the conversation transcript:

```bash
./prepper.sh --interactive --benchmark-json --interview-style behavioral_focus
```

The JSON result includes only interviewer scoring output plus run metadata, including the runtime model, benchmark scoring model, and resolved runtime model settings for comparing runs.

### Benchmark Flags Reference

- `--benchmark`: run benchmark mode with live transcript output and a bottom evaluation summary
- `-b`: short alias for `--benchmark`
- `--benchmark-json`: run benchmark mode without transcript output and print interviewer result JSON
- `--strong-candidate`: use strong candidate simulation (default)
- `--weak-candidate`: use weak candidate simulation
- `--question-limit`: max question roundtrips
- `--difficulty`: interviewer difficulty (`easy`, `medium`, `hard`) when supported by prompt
- `--pass-threshold`: override required pass score
- `--temperature`: runtime model temperature override
- `--top-p`: runtime model top-p override
- `--frequency-penalty`: runtime model frequency penalty override
- `--presence-penalty`: runtime model presence penalty override
- `--max-tokens`: runtime model max tokens override
- `--model`: model for runtime chat + candidate generation
- `--benchmark-model`: model for final interviewer scoring pass

Important: `--benchmark` and `--benchmark-json` are mutually exclusive. `--strong-candidate` and `--weak-candidate` are benchmark-only flags.

The root `./prepper.sh --benchmark` / `./prepper.sh -b` shortcut enables colored benchmark transcript output by default when the terminal supports it. Use `--color` explicitly when running the installed `prepper-cli --benchmark` command directly.

## Prompt Front Matter

Each prompt file in `src/prepper_cli/prompts/` starts with YAML front matter:

```yaml
---
id: coding_focus
name: Coding Interview
temperature: 0.3
top_p: 1.0
frequency_penalty: 0.2
presence_penalty: 0.0
max_tokens: 1200
---
```

### Settings Explained

- `id`: stable machine ID (must match filename stem)
- `name`: human-readable display label
- `temperature`: randomness/creativity
- `top_p`: nucleus sampling cutoff
- `frequency_penalty`: reduces repeated wording
- `presence_penalty`: encourages introducing new tokens/topics
- `max_tokens`: response length cap

OpenRouter compatibility note: parameters are forwarded via OpenAI-compatible API, but not all model/provider combinations honor every field.

CLI default behavior: if you do not pass a model-setting override flag, `prepper-cli` uses the selected prompt file's front matter value.

## Python API

```python
from prepper_cli import Conversation, get_chat_reply

conv = Conversation()
reply1 = get_chat_reply("Tell me about Python decorators", conversation=conv)
reply2 = get_chat_reply("Now give a short example", conversation=conv)
```

Without a `Conversation`, calls remain stateless.

You can also reconstruct history:

```python
conv = Conversation.from_messages([
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."},
])
```
