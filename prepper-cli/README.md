# prepper-cli

`prepper-cli` is the standalone Python package that powers Prepper's OpenRouter chat and interview flows.

## What It Includes

```text
prepper-cli/
|- src/prepper_cli/main.py           # CLI entry point
|- src/prepper_cli/chat.py           # get_chat_reply()
|- src/prepper_cli/conversation.py   # in-memory conversation history
|- src/prepper_cli/interview.py      # interview flow + scoring
|- src/prepper_cli/benchmark.py      # simulated benchmark interviews
|- src/prepper_cli/system_prompts.py # prompt discovery/loading
`- src/prepper_cli/prompts/           # bundled prompts (.md)
```

## Setup

```bash
cd prepper-cli
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

Optional test deps:

```bash
pip install pytest
```

Create `.env` (or export env vars):

```bash
OPENROUTER_API_KEY=your_key_here
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_MODEL=openai/gpt-4o-mini
PREPPER_DEFAULT_SYSTEM_PROMPT=coding_focus
```

## CLI Usage

From the monorepo root, you can run the same CLI via `./cli.sh` without manually activating `prepper-cli/.venv`.

### Interactive Mode (Default)

Running `prepper-cli` starts an interactive session and keeps conversation context in memory for that session.

```bash
prepper-cli
```

There is no separate `--interactive` flag.

You can preselect a system prompt:

```bash
prepper-cli --system-prompt coding_focus
```

Or list available prompt IDs:

```bash
prepper-cli --list-system-prompts
```

Interview tuning flags:

```bash
prepper-cli --system-prompt coding_focus --difficulty hard --question-limit 3 --pass-threshold 7.2
```

Model settings overrides:

```bash
prepper-cli --system-prompt coding_focus --temperature 0.2 --top-p 0.9 --frequency-penalty 0.3 --presence-penalty -0.2 --max-tokens 500
```

Language + color:

```bash
prepper-cli --color --language de --system-prompt behavioral_focus
```

Exit with `exit` or `quit`.

## Benchmark Mode (`--benchmark`)

Benchmark mode simulates a full interview automatically.

- interviewer = selected `--system-prompt`
- candidate = generated by a candidate simulation prompt
- final output includes candidate score and interviewer score

### Quick Examples

Default benchmark (good candidate profile):

```bash
prepper-cli --benchmark --system-prompt behavioral_focus
```

Force weak candidate simulation:

```bash
prepper-cli --benchmark --system-prompt behavioral_focus --bad-candidate
```

Short hard coding benchmark:

```bash
prepper-cli --benchmark --system-prompt coding_focus --difficulty hard --question-limit 3
```

Override pass threshold:

```bash
prepper-cli --benchmark --system-prompt coding_focus --pass-threshold 8.0
```

German benchmark:

```bash
prepper-cli --benchmark --system-prompt behavioral_focus --language de
```

Use different models for runtime vs final benchmark scoring:

```bash
prepper-cli --benchmark --system-prompt behavioral_focus --model openai/gpt-4o-mini --benchmark-model openai/gpt-4.1
```

### Benchmark Flags Reference

- `--benchmark`: required to run benchmark mode
- `--good-candidate`: use strong candidate simulation (default)
- `--bad-candidate`: use weak candidate simulation
- `--question-limit`: max question roundtrips
- `--difficulty`: interviewer difficulty (`easy`, `medium`, `hard`) when supported by prompt
- `--pass-threshold`: override required pass score
- `--temperature`: runtime model temperature override
- `--top-p`: runtime model top-p override
- `--frequency-penalty`: runtime model frequency penalty override
- `--presence-penalty`: runtime model presence penalty override
- `--max-tokens`: runtime model max tokens override
- `--model`: model for runtime chat + candidate generation
- `--benchmark-model`: model for final interviewer scoring pass

Important: `--good-candidate` and `--bad-candidate` are benchmark-only flags and will error without `--benchmark`.

## Prompt Front Matter

Each prompt file in `src/prepper_cli/prompts/` starts with YAML front matter:

```yaml
---
id: coding_focus
name: Coding Interview
temperature: 0.3
top_p: 1.0
frequency_penalty: 0.2
presence_penalty: 0.0
max_tokens: 700
---
```

### Settings Explained

- `id`: stable machine ID (must match filename stem)
- `name`: human-readable display label
- `temperature`: randomness/creativity
- `top_p`: nucleus sampling cutoff
- `frequency_penalty`: reduces repeated wording
- `presence_penalty`: encourages introducing new tokens/topics
- `max_tokens`: response length cap

OpenRouter compatibility note: parameters are forwarded via OpenAI-compatible API, but not all model/provider combinations honor every field.

CLI default behavior: if you do not pass a model-setting override flag, `prepper-cli` uses the selected prompt file's front matter value.

## Python API

```python
from prepper_cli import Conversation, get_chat_reply

conv = Conversation()
reply1 = get_chat_reply("Tell me about Python decorators", conversation=conv)
reply2 = get_chat_reply("Now give a short example", conversation=conv)
```

Without a `Conversation`, calls remain stateless.

You can also reconstruct history:

```python
conv = Conversation.from_messages([
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."},
])
```
